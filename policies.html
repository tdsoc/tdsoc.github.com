Policies

How to propose a new projec

Any project proposal should be submitted to the Academic Board of the TDSoC who decides of the acceptance of the new projects. To be accepted, the new project must meet four conditions:

  • it must be the implemetation of published attacks on some cryptosystem, including published papers, master theses, PhD theses, etc...
  • the cryptoanalytic algorithms must be free of any royaltes, generally speaking must be freely implementable and exploitable
  • the cryptosystem under attack must be of some practical interest (e.g.: recognized standard such as RSA, AES and E0)
  • the attack must be feasible, that is once implemented, it should run in reasonable time with commonly available hardware (e.g.: one month on a 12 cores server)

Software license

The software produced byt the TDSoC project will be released with LGPL v.3.0 license, thus, as an Open Source project.

How to partecipate

All the code and the documentation of TDSoC is freely available on the TDSoC website and can be checked out and reused for free, but see the licenses. Every project has a project leader, whom you should contact if you want to partecipate. Any software contribution by partecipant will be reviewed by the project leader before becoming official part of the TDSoC project. The roles inside each project are agreed between the project leader and the partecipants.
